# Skill: Task Tracker

Разбивает задачи на подзадачи, ведёт журнал сессий и отслеживает прогресс.

## Когда использовать

- При получении новой задачи — сразу разбить на подзадачи
- При начале работы над задачей — обновить статус
- При завершении задачи — зафиксировать результат
- Каждые 5–10 сообщений — обновить прогресс
- Перед завершением сессии — записать итог в журнал

## Где хранятся данные

Все данные хранятся в `/workspace/.qwen/tasks/` (проект-специфичная директория):

| Файл | Назначение |
|---|---|
| `active-todo.json` | Текущие задачи и их статус |
| `journal.md` | Журнал сессий (хронология) |

## Формат active-todo.json

```json
{
  "created_at": "2026-04-14T10:00:00Z",
  "updated_at": "2026-04-14T10:30:00Z",
  "tasks": [
    {
      "id": "T1",
      "title": "Описание задачи",
      "status": "pending",
      "started_at": null,
      "completed_at": null
    }
  ]
}
```

Статусы: `pending`, `in_progress`, `completed`, `cancelled`

## Алгоритм работы

### 1. Создание списка задач

При получении задачи:

1. Определи количество подзадач
2. Создай или обнови `/workspace/.qwen/tasks/active-todo.json`
3. Покажи пользователю план

### 2. Обновление статуса

При начале работы: `pending` → `in_progress`
При завершении: `in_progress` → `completed`

### 3. Журнал сессий

Каждые 5–10 сообщений или перед завершением — допиши в `/workspace/.qwen/tasks/journal.md`:

```markdown
## Сессия YYYY-MM-DD HH:MM

**Задача:** {описание}
**Выполнено:** {список}
**Осталось:** {список}
**Заметки:** {дополнительно}
```

### 4. Восстановление контекста

При старте сессии:

1. Прочитай `active-todo.json`
2. Если есть `in_progress` задачи — продолжи
3. Если файл пустой/отсутствует — working tree clean, жди задачу

## Не делай

- Не удаляй `active-todo.json` без завершения всех задач
- Не записывай секреты в файлы задач
- Не дублируй задачи — один id = одна задача
- Не смешивай разные задачи в один journal entry
