# Skill: Session State Saver

Периодически сохраняет состояние текущей сессии общения с AI-агентом на диск.

## Когда использовать

- **Автоматически:** каждые 5–10 сообщений диалога (или при явной просьбе пользователя «сохрани состояние»)
- **Перед завершением сессии:** если пользователь говорит «пока», «до свидания», «завершаю»
- **При долгом простое:** если сессия неактивна > 15 минут и возобновляется
- **После крупных изменений:** если были изменены файлы, коммиты, структура проекта

## Зачем

При обрыве связи, перезапуске контейнера или длительном простое агент теряет контекст диалога.
Сохранённое состояние позволяет:

- Восстановить контекст при возобновлении сессии
- Понять, на чём остановились, без пересказа
- Иметь историю взаимодействий для отладки

## Что сохранять

### 1. Метаданные сессии

| Поле              | Описание                                          |
| ----------------- | ------------------------------------------------- |
| `session_id`      | Уникальный идентификатор сессии (хэш даты+пути)  |
| `project_path`    | Абсолютный путь к проекту                         |
| `started_at`      | ISO-8601 timestamp начала сессии                  |
| `saved_at`        | ISO-8601 timestamp последнего сохранения            |
| `message_count`   | Количество сообщений в сессии                     |
| `model`           | Используемая модель (если известна)               |

### 2. Контекст задачи

| Поле              | Описание                                          |
| ----------------- | ------------------------------------------------- |
| `active_task`     | Текущая задача, над которой работает агент        |
| `completed_steps` | Краткое описание выполненных шагов                |
| `pending_steps`   | Что ещё предстоит сделать                         |
| `open_questions`  | Вопросы к пользователю, ожидающие ответа         |
| `decisions`       | Принятые решения и их обоснование                 |

### 3. Изменения в проекте

| Поле              | Описание                                          |
| ----------------- | ------------------------------------------------- |
| `modified_files`  | Список файлов, изменённых за сессию               |
| `created_files`   | Список созданных файлов                            |
| `deleted_files`   | Список удалённых файлов                            |
| `git_commits`     | Хэши коммитов, сделанных за сессию                |

### 4. Заметки агента

| Поле              | Описание                                          |
| ----------------- | ------------------------------------------------- |
| `notes`           | Свободные заметки агента о состоянии проекта      |
| `warnings`        | Предупреждения о потенциальных проблемах          |

## Формат хранения

JSON-файл: `/workspace/.qwen/session-state.json`

```json
{
  "session_id": "a1b2c3d4",
  "project_path": "/workspace",
  "started_at": "2026-04-10T14:30:00Z",
  "saved_at": "2026-04-10T15:00:00Z",
  "message_count": 12,
  "model": "qwen-coder",
  "active_task": "Реализовать скилл для сохранения сессии",
  "completed_steps": [
    "Изучена текущая архитектура",
    "Создан файл скилла session-state-saver.md",
    "Обновлён AGENTS.md"
  ],
  "pending_steps": [
    "Обновить README.md",
    "Обновить docs/specification.md"
  ],
  "open_questions": [],
  "decisions": [
    "Формат хранения — JSON, а не YAML, для простого парсинга"
  ],
  "modified_files": [
    "AGENTS.md",
    "config-templates/skills/session-state-saver.md"
  ],
  "created_files": [
    "config-templates/skills/session-state-saver.md"
  ],
  "deleted_files": [],
  "git_commits": [],
  "notes": "Сессия идёт продуктивно, все шаги по плану",
  "warnings": []
}
```

## Алгоритм сохранения

### 1. Подготовь данные

Собери из контекста диалога:

- Какую задачу решаешь
- Что уже сделано
- Что осталось
- Какие файлы менял
- Какие коммиты сделал (через `git log --oneline -n 5`)

### 2. Определи session_id

Если файл `.qwen/session-state.json` уже существует — возьми `session_id` оттуда.
Если нет — сгенерируй:

```bash
echo -n "$(date -u +%Y%m%d%H%M%S)-$(pwd)" | md5sum | cut -c1-8
```

### 3. Запиши файл

```bash
cat > /workspace/.qwen/session-state.json << 'STATEOF'
{ ... JSON ... }
STATEOF
```

### 4. Проверь запись

```bash
[ -f /workspace/.qwen/session-state.json ] && echo "✅ Session state saved" || echo "❌ Failed to save session state"
```

## Алгоритм восстановления

При начале сессии или после обрыва:

### 1. Проверь наличие состояния

```bash
[ -f /workspace/.qwen/session-state.json ] && echo "found" || echo "not found"
```

### 2. Если найден — прочти и восстанови контекст

```bash
cat /workspace/.qwen/session-state.json
```

Извлеки:
- `active_task` — продолжить работу
- `completed_steps` — не повторять
- `pending_steps` — что делать дальше
- `modified_files` — проверить целостность

### 3. Сообщи пользователю

```
📂 Найдено сохранённое состояние сессии:
  Задача: {active_task}
  Выполнено: {count} шагов
  Осталось: {count} шагов
  Изменено файлов: {count}
Продолжаю с того места, где остановился.
```

## Не делай

- ❌ Не сохраняй секреты, токены, ключи в `session-state.json`
- ❌ Не сохраняй личные данные пользователя
- ❌ Не удаляй файл состояния без запроса
- ❌ Не восстанавливай контекст слепо — сверяйся с текущим состоянием проекта

## Расписание

| Событие                        | Действие        |
| ------------------------------ | --------------- |
| Каждые 5–10 сообщений          | Автосохранение  |
| Пользователь: «сохрани»        | Немедленное сохранение |
| Перед завершением сессии       | Финальное сохранение |
| После `git commit`             | Обновление `git_commits` |
| После изменения файлов         | Обновление списков файлов |
